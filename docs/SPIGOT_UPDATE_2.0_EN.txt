[*][B]Plugin messaging[/B] — Bungee channel listener fixed (proper player resolution for backend “auth status” checks).
[*][B]Webhooks[/B] — optional Discord-style URLs for first registration and successful login (see [ICODE]integrations.webhooks[/ICODE]).

UltimateBungeeAuth 2.0 — major update

There are now TWO downloads: one for Waterfall/BungeeCord and one for Velocity 3. Install the jar that matches your proxy.

What's new
• Velocity 3 — login, register, queue, premium flow, admin commands
• Passwords — BCrypt; old plaintext upgraded on first successful login
• SQL — prepared statements only
• Multi-server — authServers / lobbyServers lists + lobbyPickMode (FIRST_AVAILABLE, RANDOM, ROUND_ROBIN)
• Rate limits & lockouts — per IP; optional IP bypass list
• Permissions — ultimatebungeeauth.admin (/uba), ultimatebungeeauth.queue.bypass (queue skip). Old uba.queue.bypass is gone.
• Whitelisted commands before login — editable in config
• Plugin messaging — fixed player lookup for backend auth checks
• Webhooks — optional first-register / login Discord URLs

Requirements: Java 17+ and MySQL.

Updating from 1.x
1) Backup MySQL + old config.yml
2) Remove old jar; add only Bungee OR Velocity jar (not both on same proxy for two full auth stacks without planning)
3) config.yml layout changed — use a fresh generated file and migrate values, or follow README mapping on GitHub
4) Update permissions: ultimatebungeeauth.admin and ultimatebungeeauth.queue.bypass
5) Wrong password = message + throttle by default — tune security.* if you want stricter behaviour

Build: ./gradlew build (Java 17).
